# Section Blueprints

Use these blueprints when rebuilding page flow.

## Blueprint A: Service Landing Page

1. Hero
- Eyebrow tag
- Outcome-focused headline
- One sentence subtitle
- Primary + secondary CTA
- Optional trust pills (location, response time, session format)

2. Entity Clarity Strip
- Who leads
- What focus areas
- Where/how service is delivered

3. Problem-to-Solution Cards
- 3 cards for core user needs
- Each card includes one action link

4. Process Section
- 3 step flow: assess -> plan -> follow-up
- Keep each step practical and brief

5. Trust Section
- Credentials, method clarity, boundaries
- Avoid fabricated outcomes/testimonials

6. Closing CTA
- Friction-light action with clear next step

## Blueprint B: Contact Page

1. Contact Hero
- Simple headline
- Clear expected response path

2. Quick Contact Methods
- WhatsApp
- Email
- Location/Map

3. Form
- Short, labeled fields
- Clear submit action
- Success/error handling

4. Logistics Block
- Address
- Parking/arrival notes
- Repeated direct action CTA

## Blueprint C: About Page

1. Personal Story Hero
- Why this practice exists
- Primary action into service/contact

2. Snapshot Cards
- Training
- Process style
- Clinical focus

3. Timeline / Journey
- Past -> turning point -> current method

4. Philosophy Block
- Clear treatment approach and boundaries

5. CTA
- Route to clinic or contact

## Microcopy Patterns

- CTA labels: action + destination (`Book Intro Call`, `View Clinic Process`).
- Section subtitles: one sentence, practical, no filler.
- Avoid stacked buzzwords without operational meaning.
